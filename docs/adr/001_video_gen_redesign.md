# ADR-001: video_gen.py デザイン改修

## 背景

`docs/specs/video_target_spec.md`（目標仕様）と `docs/specs/video_gen_current_spec.md`（現状仕様）の比較により、以下の点で現状の `video_gen.py` が目標仕様から乖離していることが判明した。改修の根拠は両ドキュメントに記述された事実のみに基づく。

---

## 差分一覧（優先度順）

### 🔴 高優先度（視聴者に直接影響）

#### Word Audio Mode（`generate_word_audio_video`）

- **英単語の色**:
  現状 → `"white"`（白）（current_spec 4-2, L542）
  目標 → 青色ベース＋白い縁取り（target_spec 2.1）

- **日本語訳の色**:
  現状 → `"#CCCCCC"`（グレー）（current_spec 4-2, L576）
  目標 → `"#FFFFFF"`（白）（target_spec 2.1）

- **日本語訳の配置**:
  現状 → `x=200` 固定・左寄り（current_spec 4-2, L564〜579）
  目標 → 水平中央揃え・英単語の直下（target_spec 2.1「中央下」「直下に日本語訳を配置」）

- **ロゴ位置（Word Audio Mode）**:
  現状 → 右上 `(1280 - new_w - 20, 20)`（current_spec 4-2, L289 / L514）
  目標 → 右下（target_spec 1「画面右下に固定配置」）

- **学習パートのBGM**:
  現状 → BGMあり（BGM検索→再生、current_spec 5-1 L205〜235 / L620〜641）
  目標 → **一切使用しない**（target_spec 2.3「学習パートではBGMを一切使用しません」）

#### Exam Video（`generate_exam_video`）

- **シーン末尾のQuestion Section再掲**:
  現状 → `[イントロ] → [Listening] → [Question] → [Review] → [Question（再掲）]`（current_spec 3, L1338〜1342）
  目標 → `[タイトル] → [Listening] → [Question] → [Review]`（target_spec 3.2「明確な4部構成」）

### 🟡 中優先度

#### Word Audio Mode

- **ID番号（No.xxxx）の色**:
  現状 → `"gray"`（current_spec 4-2, L520）
  目標 → 白色（target_spec 2.1）

- **単語表示時の効果音（SE）**:

- **背景テクスチャ**:
  現状 → 完全な黒（current_spec 7, L718〜730）
  目標 → 黒ベース＋微細ノイズテクスチャ（target_spec 1）

### 🟢 低優先度

- **英単語のフォントサイズ**:
  現状 → `140` 固定（current_spec 4-2, L381）
  目標 → 「画面内で最大」（target_spec 2.1）— ただし具体的な数値未定義

- **ロゴの透明度**:
  現状 → 透明度調整なし（paste 時に mask=resized_logo のみ、current_spec 4-2 L514）
  目標 → アルファ値を適切に調整（target_spec 1）— ただし具体的な値未定義

## 実装しないこと（スコープ外）

- **フェードインアニメーション**:
  目標仕様に「非常に短いフェードイン」の記載はあるが、具体的な持続時間・実装方法の記述がない。仕様が未定義のため実装しない。

- **英単語の動的サイズ自動計算（「画面内で最大」）**:
  目標仕様に「画面内で最大」とあるが、具体的な上限サイズ・余白ルールの記述がない。現状の `140` から変更する根拠が不足しているため低優先度にとどめ、具体的な数値が決まるまで実装しない。

- **複数ネイティブスピーカー音声への変更**:
  target_spec 3.3 に記載があるが、音声生成システム（`audio_gen_listening.py` 等）の変更を伴うため、`video_gen.py` 単体の改修スコープ外とする。

- **Reviewセクションでの音声同期ハイライト**:
  target_spec 3.2「音声の進行に合わせてテキストを表示またはハイライト」は、現状の静的スライド生成方式を根本から変える必要があり、別ADRで設計すべき大規模変更のため今回のスコープ外とする。

## リスク

- **#5 BGM削除**: エンドスクリーンのBGMも削除対象に含まれるか要確認。target_spec の「学習パートではBGMを一切使用しません」の「学習パート」の定義がエンドスクリーンを含むか不明確。
- **#3 日本語訳の中央揃え**: 現状の `x=200` は複数行対応を前提とした固定座標。中央揃えに変更すると意味が複数行にわたる場合に行ごとに中央計算が必要になり、`draw_centered_text_inner` の流用または新実装が必要。
- **#1 英単語の色**: 「青色ベース」の具体的な色コード（例: `#4488FF` 等）が target_spec に記載されていない。実装前に確認が必要。
- **#8 SE追加**: 使用するSEファイルのパス・ファイル名が target_spec に記載されていない。`assets/` 内の既存ファイルから選定する必要がある。
