# データ構造（SPEC）

- TARGET_CSV_PATH: vocab/ターゲット1900.csv
- カラム仕様: 「番号」「英単語」「意味」
- 範囲バリデーション: 1〜1900 を許容
- 範囲エラー時の挙動: エラーとして扱う（読込を中断）

このSPECに基づき、コードは TARGET_CSV_PATH を参照し、読み込み時にヘッダーを「番号」「英単語」「意味」に正規化する。範囲指定は 1〜1900 のみ受け付ける。 

# 東大リスニングモード難易度バランス（SPEC）

- 対象モード: university_listening（University=Todai／UTokyo）
- 目的: 「難しすぎず、しかし本番水準の思考負荷」を維持する

- 文構造
  - 参照コーパスの論理構成・段落構成は維持する
  - 1文を過度に長くしない（目安: 25〜30語程度まで）
  - 関係代名詞や挿入句の多用を避け、論点ごとに文を分割する
  - 列挙や対比は、可能な限り2文以上に分けて提示する

- 語彙と言い換え（Paraphrase）
  - 「鉄壁」でも最難関レベルの語は、その直前後で自然な言い換えを入れる
  - 言い換えは exam 単語帳以外の基本語で行い、意味上のヒントになるようにする
  - 1つの難語につき、少なくとも1回は同一段落内で言い換えを含める
  - 質問文・選択肢でも、難語の意味を推測しやすい表現を部分的に再利用する

- 音声スピードとポーズ
  - 通常の読み上げスピードを維持し、極端な高速化は行わない
  - 段落（パラグラフ）と段落の間に 0.5〜1.0 秒の無音ポーズを追加する
  - ポーズは audio_gen.py の東大モード専用ロジックでのみ挿入する
  - 受験生がメモ・要点整理に使えるよう、各段落の最後は文末で区切る

- 出題バランス
  - 質問は「内容理解」「推論」「要約」の3系統を必ず含める
  - 語彙そのものを直接問う設問と、文脈からの推測を求める設問を混在させる
  - 難易度は「正答率 50〜70%」を目安にし、奇問・パズル型は避ける
